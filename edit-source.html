写源工具 ​

1. 简介 ​

写源工具的重要性在于它极大地简化了开发者在开发源的工作流程。"喂饭式"工具，可以轻松的调试各个规则，快速完成源的编写流程。

注意

使用该工具需确保软件版本在3.3.4及以上

该工具深度融合drpy | hipy项目

道长QQ群：486722328 | 824179147

工具路径: zyplayer -> 设置 -> 写源工具

2. 教程 ​

  • 点我前往阿里云盘

3. 功能 ​

3.1. 顶栏 ​

3.1.1. 模板 ​

通过内置模板，可修改部分模板规则完成源的调试，适用于快速开发源。

如果模板一模一样甚至只需要更改url即可。

内置模板

mxpro mxone5 首图 首图2 默认 vfed 海螺3 海螺2 短视 短视2 采集 1

3.1.2. 文件 ​

  • 导入: 支持文件从外部导入，并将文件内容渲染到编辑器中，适用于调试外部文件。
  • 导出: 支持将编辑器中的内容导出为文件格式存储到本地磁盘，适用于源编辑完成或临时存储。
  • 拉取缓存: 支持将本地临时缓存内容通过后端api接口拉取到编辑器及规则中，适用于编辑器内容丢失但存在缓存的情况。

3.1.3. 运行 ​

  • 调试: 将编辑器的内容及规则提交到后端api接口存储临时缓存并生成接口，将默认站点设置为该用例，跳转至影视页面查看规则完成度和渲染效果，适用于调试用例。

  • 删除用例: 将删除本地临时缓存内容，将默认站点设置为调试前默认站点，适用于调试完成规则后使用。

  • 服务 打开该文件夹拖入文件自动加载，启用简易http服务，便于文件内容读取。

    不建议超过三层目录

    手动打开该文件夹路径

    如文件夹不存在则在以下目录上一级手动创建 file 文件夹

    Windows: %USERPROFILE%\AppData\Roaming\zyplayer\file\

    Macos: ~/Library/Logs/zyplayer/file/

    Linux: ~/.config/zyplayer/file/

    • 单文件

    访问地址：http://127.0.0.1:9978/api/v1/file/文件名

    如: 文件11.js为11.js; tmp/11.js为/tmp/11.js

    • 一键

    1.访问地址：http://127.0.0.1:9978/api/v1/file/config此方法递归读取文件夹下符合规则的文件，不建议超过三层目录。

    2.访问地址：http://127.0.0.1:9978/api/v1/file/生成式路径地址

    何为生成式

    即提供的本地包中包含index.js文件, 该文件需包含mian函数。

    生成式路径地址为本地包路径, 无需添加/index.js, 添加则代表读取读取index.js文件。

    访问生成式时会注入pathLib对象,调用main函数，并返回结果。

    javascript
    // 调用方法示例 `pathLib.join`
pathLib = {
  join: path.join,
  dirname: path.dirname,
  readDir: fs.readdirSync,
  readFile: fs.readFileSync,
  stat: fs.statSync,
}

3.2. 工作区 ​

3.2.1. 源代码 ​

输入url，通过请求该地址获取源代码，将于源代码中输出，方便查看源代码dom结构，适用于源代码的获取。

请求参数说明

点击 ⬛️ 设置

响应编码: 支持UTF-8|GBK|GB2312|GB18030, 默认为UTF-8

请求头(字典类型): 示例如下

json
{ "User-Agent": "zyplayer" }

请求方法: 除GET方法外生效，支持application/json|application/x-www-form-urlencoded

请求体(字典类型): 除GET方法外生效，示例如下

json
{ "key": "01b97b" }

3.2.2. 列表 ​

输入pdfa规则，匹配源代码内容，将于规则中输出，方便查看匹配结果，输出结果为列表，适用于pdfa调试。

注意

必须先获取源代码，否则不生效

3.2.3. 节点 ​

输入pdfh规则，匹配源代码内容，将于规则中输出，方便查看匹配结果，适用于pdfh调试。

注意

必须先获取源代码，否则不生效

3.2.4. 嗅探 ​

输入url，根据系统设置的嗅探模式进行获取，将获取的结果在调试中输出，适用于嗅探的获取。

注意

需设置系统嗅探模式

如设置嗅探参数可以更好完成匹配

嗅探需要时间并不能马上完成, 一般情况下最长为 1 分钟。

嗅探参数说明

点击 ⬛️ 设置

辅助规则(正则): 赋值嗅探提取参数，示例如下

txt
obj/tos|bd.xhscdn|/ugc/

初始化脚本(js): 嗅探前设置初始化变量, 常用于规避浏览器特性，示例如下

javascript
Object.defineProperties(navigator, {platform: {get: () => 'iPhone'}});

执行脚本(js): 嗅探是进行的操作, 常用于某些元素的点击，示例如下

javascript
document.querySelector(".line").click();

3.2.5. 编辑器 ​

最主要内容，用于编辑源代码。

编辑器参数

主题: Dark | Light

语言: json | yaml | xml | javascript | java | python

行位符: LF | CRLF

自动换行: 启用 | 禁用

提示

只是想尝试一下？跳到写源语法

编辑器支持ctrl+f搜索关键词

3.2.6. 初始化 ​

根据编辑器内容初始化源代码，初始化结果将于调试输出, 同时过程中的日志也会于日志输出, 适用于初始化调试。

事件说明

  1. 点击👋 触发事件: 切换模式[自动 | 手动]

  2. 点击除👋外 触发事件: 初始化

模式说明

  1. 当模式为自动时, 编辑器内容最后修改时间＞上次初始化时间, 触发分类、首页、列表、详情、搜索、搜索、代理动作时自动初始化, 无需手动再次初始化。

  2. 当模式为手动时, 编辑器内容每次修改后, 触发分类、首页、列表、详情、搜索、搜索、代理动作之前均需再次手动初始化。
